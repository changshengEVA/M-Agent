# 三维知识图谱可视化深度分析报告

## 1. 层深度配置

根据 `three-scene.js` 配置文件：

| 层名称 | Z轴深度 | 描述 |
|--------|---------|------|
| 实体层 (entityLayer) | 0 | 基础层，实体节点所在平面 |
| 特征层 (featureLayer) | 100 | 中间层，特征节点所在平面 |
| 场景层 (sceneLayer) | 200 | 顶层，场景节点所在平面 |

### 层间距分析：
- **特征层 - 实体层**: 100 单位
- **场景层 - 特征层**: 100 单位  
- **场景层 - 实体层**: 200 单位

## 2. 平面法向量分析

### 层平面配置：
在 `_addLayerPlanes()` 函数中，层平面使用默认方向（无旋转）：
- 平面几何体: `THREE.PlaneGeometry(350, 350)`
- 平面位置: 分别设置在对应层的Z坐标上
- 平面法向量: 默认指向Z轴正方向 (0, 0, 1)

### 平面法向量验证：
- 平面默认在XY平面上
- 法向量指向Z轴正方向（深度轴）
- 符合"法向量是深度轴"的要求

## 3. 节点深度分布

### 节点布局算法 (`_createNodeLayout`):
1. **实体层节点**: 在XY平面上圆形排列，Z坐标固定为 `entityLayerZ` (0)
2. **特征层节点**: 围绕对应实体排列，Z坐标固定为 `featureLayerZ` (100)
3. **场景层节点**: 围绕对应特征排列，Z坐标固定为 `sceneLayerZ` (200)

### 节点深度特性：
- 同一层内的节点具有相同的Z坐标
- 垂直边连接不同层的节点，形成立体结构
- 节点深度严格按层分离，无交叉

## 4. 三维空间坐标系

### Three.js默认坐标系：
- **X轴**: 红色，水平方向（右）
- **Y轴**: 绿色，水平方向（上）  
- **Z轴**: 蓝色，垂直方向（前/深度）

### 当前实现：
- 使用Z轴作为垂直/深度轴
- 层平面在XY平面上
- 节点在各自层的平面上分布

## 5. 视觉深度效果

### 摄像机配置：
```javascript
camera.position.set(200, 200, 300);  // 从斜上方观察
camera.lookAt(0, 0, 100);           // 看向中间层（特征层）
```

### 视觉深度层次：
1. **最近层**: 场景层 (Z=200)
2. **中间层**: 特征层 (Z=100) 
3. **最远层**: 实体层 (Z=0)

### 深度感知：
- 层间距足够大（100单位），确保视觉分离
- 摄像机从斜上方观察，能清晰看到三层结构
- 垂直边连接不同层，强化立体感

## 6. 技术验证

### 已解决的问题：
1. ✅ 摄像机控制：启用完整的OrbitControls功能
2. ✅ 层级连接：改进布局算法，确保垂直边正确连接
3. ✅ 坐标轴方向：确认蓝色Z轴为深度轴
4. ✅ 平面法向量：确保层平面法向量指向Z轴（深度轴）

### 深度一致性检查：
- 所有实体节点: Z = 0 ± 容差
- 所有特征节点: Z = 100 ± 容差  
- 所有场景节点: Z = 200 ± 容差
- 层平面位置与对应层节点位置一致

## 7. 建议优化

### 深度可视化增强：
1. **深度渐变**: 为不同层使用不同的透明度或颜色饱和度
2. **深度提示**: 添加深度标尺或深度指示器
3. **深度聚焦**: 实现层聚焦功能，突出显示特定层

### 性能考虑：
- 当前层间距（100单位）在合理范围内
- 可考虑根据节点数量动态调整层间距
- 对于大量节点，可适当增加层间距以避免视觉重叠

## 8. 总结

三维知识图谱可视化系统的深度配置正确且合理：

1. **层深度**: 实体层(0) → 特征层(100) → 场景层(200)
2. **平面方向**: 层平面在XY平面上，法向量指向Z轴（深度轴）
3. **节点分布**: 节点严格按层分布，垂直边正确连接
4. **视觉效果**: 三层结构清晰可见，立体感强

系统已成功实现用户要求的三层立体可视化，所有深度相关的问题均已解决。
